# XHS / Douyin Enhancement

用于说明小红书与抖音的增强访问模式。默认模式仍应优先使用公开搜索与多平台交叉验证。

## 环境变量

可选：
- `XHS_COOKIE`
- `DOUYIN_COOKIE`

用途：
- 提升公开页面读取成功率
- 提升搜索结果页或分享页的稳定性
- 帮助提取更多经验层信息

限制：
- 不保证可访问私有内容
- 不保证长期稳定
- 不应作为唯一数据源

## 什么时候启用

在以下场景优先考虑启用增强模式：
- 用户明确要求优先搜索小红书 / 抖音
- 默认公开搜索结果过少
- 页面存在登录墙但用户接受使用自己的登录态
- 需要补充“近期热点”“出片机位”“短视频热词”等经验层信息

## 什么时候不要硬用

以下情况不要把增强模式当刚需：
- 只需要开放时间、票价、预约规则
- 只需要 POI、距离、路线
- 国内基础吃喝玩乐已经可由高德 / 大众点评 / 官方页满足
- 平台风控明显且继续尝试只会浪费时间

## 小红书建议提取内容

适合提取：
- 高频景点
- 高频避坑词
- 高频预约提醒
- 店铺热词
- 出片点与机位
- 路线经验

不适合单独决定：
- 官方开放时间
- 实时票价
- 闭馆公告
- 精确交通时间

## 抖音建议提取内容

适合提取：
- 近期热门玩法
- 夜景路线
- 打卡点
- 视频热词
- 热门店铺与玩法趋势

不适合单独决定：
- 营业时间
- 预约规则
- 票价
- 酒店实时价格

## 失败回退策略

增强模式失败时，自动回退到：
1. 搜索引擎索引页
2. 高德 / Google Maps
3. 大众点评 / 马蜂窝 / 携程 / 飞猪
4. 官方景区 / 文旅 / 博物馆页面

## 结论原则

- 小红书 / 抖音主要是“内容发现层”和“经验层信号源”。
- 地图与官方页面才是“事实层”。
- 规划结果必须把两者区分开，不要把单帖经验伪装成硬事实。
