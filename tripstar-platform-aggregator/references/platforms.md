# Platforms Reference

按目的地与任务选择平台，不追求全覆盖，追求稳定、可交叉验证。

## 平台清单与用途

### 1. 小红书
- 强项：出片机位、真实踩坑、预约提醒、路线经验、美食踩雷
- 弱项：登录墙、风控、页面变化快
- 用法：优先公开页面；提取高频景点/高频踩坑词；与地图和官方页交叉验证

### 2. 抖音
- 强项：近期热点、夜景、美食、玩法动态、视频感知强
- 弱项：动态加载多、信息碎片化、网页可访问性不稳定
- 用法：提取近期热度与玩法，避免单独作为事实来源

### 3. 高德地图
- 强项：国内 POI、营业信息、顺路关系、商圈聚类、餐饮店铺
- 弱项：长文攻略弱
- 用法：国内路线设计优先；用于区域落点和同日串联

### 4. Google Maps
- 强项：境外 POI、评论、营业时间、路线与地理关系
- 弱项：国内部分数据不如高德
- 用法：境外优先；国内作为补充交叉验证

### 5. Google Travel
- 强项：境外旅行概览、热门点、季节性认知
- 弱项：不一定覆盖国内深度玩法
- 用法：做境外大盘 overview 与路线草图

### 6. 飞猪
- 强项：住宿、门票、玩乐产品、国内预订场景
- 弱项：攻略内容结构化一般
- 用法：看价格带、住宿区域、景区门票参考

### 7. 携程
- 强项：酒店/景点/交通覆盖广，国内行程价格参考强
- 弱项：UGC 真实性需交叉验证
- 用法：住宿价格带、交通时间、热门景点基础信息

### 8. Booking / Agoda
- 强项：境外住宿横向价格与住客评价
- 弱项：中文目的地攻略弱
- 用法：境外住宿区域和预算校准

### 9. 大众点评
- 强项：吃喝、本地店铺、口味和排队信息
- 弱项：景点攻略能力弱
- 用法：餐饮筛选；避开纯流量店；与小红书交叉验证

### 10. 马蜂窝
- 强项：经典长线攻略、路线总览、目的地百科感强
- 弱项：实时性一般
- 用法：作为传统攻略底稿，帮助快速建立城市结构认知

### 11. Tripadvisor
- 强项：境外景点/餐饮评论、国际游客视角
- 弱项：国内覆盖深度有限
- 用法：境外目的地交叉验证评论与景点优先级

### 12. YouTube
- 强项：境外视频攻略、vlog、路线体验
- 弱项：检索成本较高
- 用法：补充视频感知与近期玩法，尤其境外

### 13. Bilibili
- 强项：中文长视频攻略、路线讲解、避坑经验
- 弱项：结构化程度较弱
- 用法：国内目的地深度补充，适合长线/城市 walkthrough

### 14. Rome2Rio
- 强项：跨城交通方式与大致时间关系
- 弱项：价格仅供参考
- 用法：城市间移动或跨国段交通比较

### 15. 官方页面
- 强项：开放时间、票价、预约规则、展馆公告、临时闭馆
- 弱项：体验信息不足
- 用法：所有“需不需要预约/几点开门/是否闭馆”优先以官方为准

### 16. 天气源
- 强项：天气、气温、降雨
- 用法：决定是否安排夜景、户外、公园、爬山、博物馆替代方案

## 按任务的推荐组合

### 国内城市休闲游
- 小红书 + 高德 + 大众点评 + 官方页 + 天气

### 国内亲子/老人同行
- 高德 + 官方页 + 马蜂窝 + 大众点评 + 天气

### 国内网红出片/美食游
- 小红书 + 抖音 + 高德 + 大众点评

### 境外城市经典游
- Google Maps + Google Travel + Booking + Tripadvisor + 官方页 + 天气

### 境外深度体验游
- Google Maps + YouTube + Tripadvisor + Booking + Rome2Rio

## 平台冲突处理

- 店铺很火但点评稳定性差：降权
- 小红书说需预约但官方未写：标注“疑似需预约，建议出发前复核”
- 地图显示营业中但官方公告闭馆：以官方为准
- 多平台都提到的景点但距离太散：拆分到不同天
